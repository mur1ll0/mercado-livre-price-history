# AGENTS.md

## Project Overview

Mercado Livre price tracker with AI-based product deduplication. Node.js/Express + MongoDB + Puppeteer scraper. Deployed on Vercel with GitHub Actions cron.

## Commands

```bash
npm run dev           # Start local server (node src/server.js)
npm run test-scraper  # Test scraper against hardcoded URLs
npm start             # Same as dev (production start)
```

No build step, no lint, no typecheck, no test framework.

## Entry Points

- **Local dev:** `src/server.js` — loads dotenv, connects DB, starts Express on `PORT` (default 3000)
- **Vercel:** `api/index.js` — serverless handler wrapping `src/app.js`
- **Cron/scraper:** `src/cron-scraper.js` — can run standalone (`node src/cron-scraper.js [announcementId] [fallbackUrl]`) or imported

## Environment Variables

| Variable | Notes |
|---|---|
| `MONGODB_URI` | Takes precedence over `MONGO_URI` in local code |
| `MONGO_URI` | Used by GitHub Actions workflow secret |
| `JWT_SECRET` | Has hardcoded fallback `'fallback-jwt-secret-key-local'` — works in dev without setting |
| `GOOGLE_CLIENT_ID` | Required for auth |
| `OPENROUTER_API_KEY` | Required for AI product matching |
| `GITHUB_PAT` / `GITHUB_REPO` | Only needed on Vercel to trigger Actions via dispatch API |

## Puppeteer / Chromium

- `.puppeteerrc.json` skips automatic Chrome download (`skipDownload: true`)
- Must install Chromium manually: `npx puppeteer browsers install chrome`
- GitHub Actions CI installs it and `libgbm-dev` system deps
- Launches with `headless: 'shell'`, stealth plugin, custom UA (Chrome 120), request interception blocking fonts/media/analytics

## MongoDB DNS Quirk

`src/db.js` forces IPv4 DNS (`dns.setDefaultResultOrder('ipv4first')` + Google DNS servers). This matters when connecting to MongoDB Atlas from Windows — without it, IPv6 resolution may fail.

## Architecture Notes

### Dual Execution (Hybrid Scraping)

- **Vercel (production):** POST to `/api/products/track` triggers GitHub Actions via repository dispatch API; returns HTTP 202 immediately
- **Local dev:** same endpoint dynamically imports the scraper and runs it in-process as a background promise (non-blocking)

The check is `if (!process.env.VERCEL)` in the `triggerGitHubScraper()` helper — Vercel sets this env var automatically.

### Product Deduplication (`src/services/ai-matcher.js`)

Two-tier matching when a newly scraped listing arrives:
1. Jaccard text similarity against all products in the same top-level category
2. If similarity is 0.35–0.85 (ambiguous), calls OpenRouter LLM to compare titles

**Important:** The actual model used is `meta-llama/llama-3-8b-instruct:free` (free tier), NOT `google/gemini-2.5-flash` as the README states. Changing models affects cost and response format.

### Mongo Model IDs

- `Announcement._id` — string (MLB/MLBU IDs, e.g. `MLB54106888`)
- `UnifiedProduct._id` — string (`crypto.randomUUID()` or temp prefix like `temp-<timestamp>`)
- `Category._id` — string (full path, e.g. `"Celulares e Telefones > Acessórios"`)
- `User._id` — ObjectId (auto-generated by Mongoose)
- Temporary product IDs use format `temp-${Date.now()}-${random}` and get resolved/replaced on first successful scrape

### Catalog Pages

The scraper detects catalog URLs (`/p/MLB...`) and scrapes TWO variations:
1. The base URL
2. Same URL with `?offer_type=BEST_INSTALLMENTS` appended

Both results are stored as separate Announcements (suffixed `_BEST_INSTALLMENTS`). This is why `scrapePage` returns an array for catalog pages.

### Rate Limiting

`src/cron-scraper.js:168` — hardcoded 4-second delay between announcements via `setTimeout(resolve, 4000)`.

### Scraper Bot Detection

`src/scraper.js` checks for captcha/block indicators (account verification URL, "não sou um robô", title = "Mercado Livre", trace-id). If detected, throws `Error('Blocked by Mercado Livre bot protection')`.

## Scraper Debugging

`scratch_inspect_links.js` — standalone script to inspect a URL's HTML content, counts Cheerio selectors, and dumps first 500 chars of body text. Useful for debugging when ML changes their markup.

## Category Tree

Categories are saved hierarchically in the `Category` collection using the full breadcrumb path as `_id`. The `/api/categories` endpoint returns all nodes sorted, and `/api/products/ranked` supports `?category=` filtering via regex prefix matching on the category path.
